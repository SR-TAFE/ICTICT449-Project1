What is the GIT
3 State Model

  • Working Directory
  • Staging Area
  • Repository

Working Directory

The Working Directory is where your project's files reside. It's your workspace for making changes. Here, you create, edit, and delete files, and Git monitors these modifications.

Staging Area

The Staging Area is a crucial intermediate step. It acts as a selective snapshot of your changes. Before committing, you add specific files or changes to the stage. This allows you to review and organize your updates.

Repository

The Repository is the Git database. It stores all versions of your project. When you commit your staged changes, Git creates a new snapshot in the repository. It enables you to track your project's history and collaborate effectively with others.

Why Should You Use Git?

  • Version Control: Git allows you to track changes in your project over time.
  • Collaboration: Git facilitates collaboration among team members.
  • Branching: Git enables you to create branches for independent development.
  • Backup and Recovery: Git acts as a robust backup system.
  • Code Review: Git makes it easy to review and comment on changes.
  • Open Source: Git is open-source with a vast community of users and developers.
  • Distributed System: Git is distributed, allowing offline and remote work.
  • Scalability: Git scales well for both small and large projects.
  • Integration: Git can be integrated with various development tools.
  • Flexibility: Git can version any type of files, not limited to code.
